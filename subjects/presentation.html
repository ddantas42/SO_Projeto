Arquitetura do Sistema

1 Processo Principal + 2 Processos Filhos (fork) + 5 Threads

Main Process

fork() × 2. Cria shared memory, aguarda filhos com waitpid()

shmget() + fork() + waitpid()

Child: Exploração

3 threads drone. sleep(5s), deposita amostras

#define NUMBER_OF_DRONES_THREADS 3

Child: Análise

2 threads braço. nanosleep(1-3s), registra CSV

Estrutura Compartilhada (Real)

t_sharedboard — dados do IPC

typedef struct s_sharedboard

{

t_sample samples[STORAGE_CAPACITY]; // 10 slots

int in; // índice entrada (drone)

int out; // índice saída (análise)

int count; // quantas amostras tem

pthread_mutex_t board_mutex;

pthread_mutex_t log_mutex;

volatile sig_atomic_t stop_signal; // SIGINT (Ctrl+C)

volatile sig_atomic_t pause_analysis; // SIGTSTP (Ctrl+Z)

} t_sharedboard;

shmget/shmat:

Main cria, ambos filhos referenciam

Circular Buffer:

in/out com modulo STORAGE_CAPACITY (10)

Sincronização

Mutex + Busy-wait com sleep() — Sem esperas ativas pesadas

Mecanismos

  • board_mutex — protege acesso ao tabuleiro
  • log_mutex — protege output sincronizado
  • count + sleep() — verificação com retry (busy-wait reduzido)

Padrão Real

// Drone (Produtor)

pthread_mutex_lock(&board->board_mutex);

if (board->count >= STORAGE_CAPACITY) {

pthread_mutex_unlock(&board->board_mutex);

logger(DRONE_LOG, id, "Waiting for space...");

sleep(1);

// retry no próximo loop...

}

deposit_sample(board, &sample);

pthread_mutex_unlock(&board->board_mutex);

// Braço (Consumidor)

pthread_mutex_lock(&board->board_mutex);

if (board->count <= 0) {

pthread_mutex_unlock(&board->board_mutex);

logger(ANALYSIS_LOG, id, "Waiting for samples...");

sleep(1);

// retry no próximo loop...

}

local_sample = get_sample(board);

pthread_mutex_unlock(&board->board_mutex);

Sinais (Signals)

Controlo do sistema via SIGTSTP e SIGINT

Ctrl+Z

SIGTSTP — Toggle Análise

Ativa/desativa o módulo de análise. Quando desativado, braços entram em standby (bloqueados num semáforo ou flag condicional).

signal(SIGTSTP, toggle_analysis_handler);
Ctrl+C

SIGINT — Terminação Limpa

Termina todos os processos e threads. Liberta shared memory (shmdt + shmctl), destrói semáforos e mutex.

signal(SIGINT, cleanup_handler);

Considerações-Chave

Requisitos críticos do projeto

Sem Esperas Ativas

Usar sleep() para reduzir CPU com busy-wait. Mutex protege acesso crítico.

Race Conditions

Mutex protege toda operação no tabuleiro. Acesso atómico garantido.

Cleanup

shmdt, shmctl, sem_destroy, pthread_join — tudo no handler de SIGINT.

MAX_SAMPLES=10, NUM_DRONES=3, COLLECT_TIME=5 — fácil configuração.
